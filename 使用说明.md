## 安装与启动

### 安装方式

**方式 A：MSI 安装包（推荐 Revit 2025-2027 用户）**
1. 双击 `EkahauWiFiTools-v2.0.0.msi`
2. 自动安装到 `C:\ProgramData\Autodesk\Revit\Addins\{版本}\`
3. 启动 Revit

**方式 B：PowerShell 脚本（覆盖 Revit 2023-2027）**
```powershell
# 默认安装到当前用户目录
.\install.ps1

# 安装到所有用户（需管理员权限）
.\install.ps1 -Scope AllUsers
```
脚本会自动检测已安装的 Revit 版本并将匹配的 DLL 部署到对应文件夹。

### 启动后的位置

打开 Revit → 顶部出现 **WiFi Tools** 选项卡，包含两个面板：

| 面板 | 按钮 |
|------|------|
| **Export & Read** | Param Config / ESX Export / ESX Read |
| **Access Point** | AP Place |

每个按钮都有自己的图标和悬停提示。

## 推荐工作流程

```
┌──────────────┐  ┌─────────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│ Param Config │→ │ ESX Export  │→ │ Ekahau   │→ │ ESX Read │→ │ AP Place │
│ 配置墙体材料 │  │ 导出楼层图  │  │ 设计     │  │ 导入 AP  │  │ 放置族   │
└──────────────┘  └─────────────┘  └──────────┘  └──────────┘  └──────────┘
   一次性配置        每个视图           外部           导入临时         正式
                    导出一次         WiFi 设计         标记预览       放置族实例
```

**典型时间线：**
1. 项目初次接入 Ekahau → 跑一次 **Param Config**
2. 对要做无线规划的每个楼层视图 → 跑一次 **ESX Export**
3. Ekahau Pro 中完成 AP 设计 → 保存为 .esx 文件
4. **ESX Read** 把 AP 位置导回 Revit（先以临时标记预览）
5. **AP Place** 用真实族实例替换预览标记，并自动生成 WiFi 平面视图和 AP 明细表

## 1. Param Config — 参数配置

### 用途

在 Wall（墙）、Door（门）、Window（窗）的**类型**上写入 `Ekahau_WallType` 共享参数。这个参数在 ESX Export 时被读取，决定每段墙在 Ekahau 中使用哪种 RF 衰减预设（如 `BrickWall`、`GlassWall`、`Concrete` 等）。

### 前置条件

- 当前活动视图必须是**楼层平面视图**
- 视图中至少包含一个墙、门或窗实例

### 操作步骤

1. **打开你想配置的楼层平面视图**

2. **点击 `WiFi Tools → Param Config`**

3. **共享参数自动创建**
   - 首次运行：在 `%APPDATA%\Ekahau_SharedParams.txt` 创建共享参数文件
   - 自动绑定 `Ekahau_WallType` 到墙/门/窗类别
   - 弹出确认对话框

4. **（可选）选择链接模型**
   - 如果项目中链接了其他 Revit 模型（如结构模型），会弹出"链接模型选择"对话框
   - 选择要一起配置的链接模型，或选择"仅主体模型"

5. **类型映射对话框**
   - 显示当前视图中所有的墙/门/窗**类型**（不是实例，每种类型只列一行）
   - 每行显示：类型名称、类别、当前值、建议预设、来源（参数/关键字/默认）
   - **建议预设的来源**：
     - `Param: xxx` — 已经设置过的值
     - `Name match: 'concrete'` — 通过类型名称中的关键字匹配
     - `Material match: 'glass'` — 通过复合结构层中的材料名匹配
     - `Curtain Wall detected` — 自动识别为幕墙
     - `Default for wall/door/window` — 默认值

6. **修改下拉框选择 Ekahau 预设**
   - 每行的下拉框可以选择 `Skip`（跳过不写）或具体预设
   - 链接模型中的类型会标注 `[Link]`，配置会保存在 ExtensibleStorage 中（不修改链接源文件）

7. **点击"确定"应用**
   - 主体模型类型 → 写入 `Ekahau_WallType` 参数
   - 链接模型类型 → 写入 ExtensibleStorage（专属本项目）
   - 弹出汇总对话框：更新数 / 跳过数 / 总数

### 关键点

- **只需要做一次**。后续修改新增的墙类型时再跑一遍即可
- 跳过 (Skip) 的类型在 ESX Export 时会用默认预设处理
- 类型名称里如果包含 `concrete`、`brick`、`glass`、`drywall` 等关键字，系统会自动建议匹配的预设

## 2. ESX Export — 导出到 Ekahau

### 用途

把当前 Revit 项目的楼层平面图（包括墙/门/窗几何）导出为 Ekahau Pro 可以打开的 `.esx` 文件。Ekahau 工程师在此基础上做无线规划。

### 前置条件

- 已完成 **Param Config**（推荐，否则所有墙都使用默认预设）
- 项目中至少有一个楼层平面视图，且视图启用了**裁剪框**（CropBox Active）

### 操作步骤

1. **点击 `WiFi Tools → ESX Export`**

2. **视图选择对话框**
   - 列出项目中所有启用了裁剪框的楼层平面视图
   - 用复选框选择要导出的视图（可多选）
   - 顶部有搜索框可按名称过滤

3. **导出模式选择**
   - **Merge All**（合并）：把所有选中的视图合并成一个 `.esx` 文件，每个视图作为一层楼
   - **Separate**（分离）：每个视图导出为单独的 `.esx` 文件

4. **链接模型选择**
   - 与 Param Config 类似，可以把链接模型中的墙/门/窗一起导出
   - 选择链接模型 → 系统会读取该链接的 ExtensibleStorage 映射

5. **逐视图的映射检查**
   - 每个视图导出前都会再次确认墙类型 → Ekahau 预设的映射
   - 如果当时的视图含有未在 Param Config 配置过的类型，会高亮提醒
   - 三个按钮：
     - `Export` — 接受当前映射，继续导出
     - `Skip View` — 跳过这个视图，处理下一个
     - `Cancel All` — 整个流程中止

6. **保存对话框**
   - 选择 `.esx` 输出位置（Merge 模式）或文件夹（Separate 模式）

7. **导出过程**
   - PNG 渲染（视图导出为图像）
   - 几何提取（墙端点、门窗位置在世界坐标系下的坐标）
   - 单位换算（英尺 → 米，像素 → 米）
   - 打包为 ZIP 格式的 `.esx`

8. **完成对话框**
   - 显示输出路径、视图数量、墙/门/窗段数量

### 关键点

- 视图的**裁剪框范围**决定导出图像的范围；范围太大会导致 PNG 巨大、Ekahau 加载慢
- **视图比例**影响导出图像的清晰度（建议 1:100 或 1:200）
- 旋转过的视图会自动转换坐标到正北方向

## 3. ESX Read — 从 Ekahau 导入 AP

### 用途

读取 Ekahau Pro 完成 WiFi 规划后保存的 `.esx` 文件，把 AP 位置以**临时标记**形式预览到 Revit 视图中（不放真实族），并写入暂存数据供 AP Place 使用。

### 前置条件

- 有一个 Ekahau 设计的 `.esx` 文件
- 该 `.esx` 中的楼层名称最好与 Revit 视图名称对应（不严格要求，可手动匹配）

### 操作步骤

1. **点击 `WiFi Tools → ESX Read`**

2. **选择 .esx 文件**
   - 标准的文件选择对话框

3. **解析进度**
   - 后台解压 .esx → 读取 floorPlans / accessPoints / radios / antennaTypes 等 JSON
   - 通常 1-3 秒完成

4. **楼层匹配对话框**
   - 左列：.esx 中的 Floor Plan 名称
   - 右列：Revit 中的 ViewPlan 下拉框
   - 自动尝试三种匹配：
     1. **完全相同**（忽略大小写）
     2. **包含匹配**（一方包含另一方且唯一）
     3. **未匹配** → 显示 `(请选择)`
   - 手动调整需要的下拉框
   - 取消勾选不想处理的楼层

5. **AP 确认对话框（每个楼层）**
   - 列出该楼层上所有的 AP，包含：
     - 名称、厂商、型号
     - 频段、坐标 (X, Y)
     - 安装高度（米）
   - 用复选框选择要预览的 AP
   - 顶部有"全选/全不选"按钮

6. **临时标记自动放置**
   - 在选中的视图中放置 **DetailLine + TextNote**（蓝色十字 + AP 名称）
   - 标记带有项目隔离的元数据，可以被后续 ESX Read 安全识别和清理
   - **自动清理上次的标记**（Bug Fix #10）— 重复运行不会产生重复标记

7. **写入暂存 JSON**
   - 路径：`%TEMP%\EkahauRevitPlugin\{项目名}_{哈希}\ap_staging.json`
   - 包含每个 AP 的世界坐标、安装高度、12 项 Radio 设计数据汇总（厂商、型号、频段、技术、TxPower、信道、空间流、天线信息等）
   - 每个项目都有独立的目录（基于文档路径的 MD5 哈希），多项目并行不会互相干扰

8. **完成对话框**
   - 显示：放置的标记数 / 楼层数 / 暂存文件路径
   - 提示用户运行 AP Place 完成后续放置

### 关键点

- **临时标记是预览性质**，可以随时手动删除
- **多次运行 ESX Read** 会自动清理上次的标记，重新放置
- 标记带有项目专属哈希，不同项目的标记不会互相影响
- 暂存数据持久化，关闭 Revit 后再打开，AP Place 仍能找到

## 4. AP Place — 放置 AP 族实例

### 用途

读取 ESX Read 写入的暂存数据，让用户选择一个 AP 族类型，把所有 AP 放置为**真实的 Revit 族实例**，写入完整的 Ekahau 参数，并可选生成 WiFi 平面视图和 AP 明细表。

### 前置条件

- 已运行过 **ESX Read**（同一项目，否则提示项目不匹配）
- 项目中已加载至少一个 AP 族（推荐"Generic Models"类别下的 WiFi AP 族）

### 操作步骤

#### 步骤 1：启动

点击 `WiFi Tools → AP Place`

#### 步骤 2：暂存数据校验
- 系统检查 `%TEMP%` 中的 ap_staging.json
- 校验项目路径哈希是否匹配（防止跨项目误用）
- 校验视图是否还存在（重命名/删除的会被跳过并提示）

#### 步骤 3：族选择器（三列联动）

- **左列**：Category（类别）— Generic Models / Electrical Equipment 等
- **中列**：Family（族名称）
- **右列**：Type（类型）
- 顶部搜索框可跨三列过滤
- 系统会**记住上次的选择**，再次运行时自动定位

#### 步骤 4：自动创建 12 项 Ekahau 共享参数

- `Ekahau_AP_Name`、`Ekahau_Vendor`、`Ekahau_Model`
- `Ekahau_Mounting`、`Ekahau_Bands`、`Ekahau_Technology`
- `Ekahau_TxPower`、`Ekahau_Channels`、`Ekahau_Streams`
- `Ekahau_Antenna`、`Ekahau_Tags`
- `Ekahau_MountHeight_m`（数值类型）
- 全部以 **InstanceBinding** 绑定到所选族的类别

#### 步骤 5：放置确认对话框

- 按楼层分组列出所有未放置的 AP
- 每行有复选框，可单独排除某些 AP
- 工作集下拉框（如果项目启用了工作共享）
- "清理预览标记"开关（默认勾选）

#### 步骤 6：分批放置（每批 20 个 AP，避免大事务）

- 每个 AP：
  - 在视图所在标高放置 FamilyInstance
  - 设置 Z = 安装高度（来自 ESX 数据）
  - 写入 `Mark = AP 名称`
  - 写入 `Comments = 厂商 | 型号 | 频段 | 高度 | 标签`
  - 写入 12 项 Ekahau 共享参数
  - 设置工作集（如果选择了）

#### 步骤 7：清理预览标记

- 删除 ESX Read 创建的 DetailLine + TextNote
- 三层 try/catch 保护，不会因为一个标记失败影响整批

#### 步骤 8：放置结果汇总

- 显示总放置数 / 失败数 / 每楼层放置数
- 列出警告（如某个 AP 放置失败的原因）

#### 步骤 9：（可选）创建 WiFi 平面视图

弹窗询问"是否创建 WiFi 平面视图？"
- **Yes** → 每个有 AP 的楼层创建一个独立视图：
  - 视图名 `WiFi Plan - {标高名}`
  - **只显示**：墙、门、窗、柱、楼梯、楼板（半色调）、网格、放置的 AP
  - **隐藏**：所有家具、设备等无关元素
  - **AP 高亮**：每个 AP 用蓝色加粗边线标记
  - **AP 标签**：每个 AP 上方放置 TextNote 显示其 Mark 名称（颜色与 AP 边线匹配）
  - 标签字体大小自动匹配视图比例
  - **过滤器**：自动创建 `Ekahau - Hide Non-AP Elements` 视图过滤器，用 `Ekahau_AP_Name 无值` 规则隐藏 Generic Models 中的非 AP 元素
- **No** → 跳过

#### 步骤 10：（可选）创建 AP 明细表

弹窗询问"是否创建 AP 明细表？"
- **Yes** → 每个有 AP 的标高创建一个明细表：
  - 表名 `Ekahau AP Schedule - {标高名}`
  - **过滤器**：`Ekahau_AP_Name 有值` + `Level = 当前标高` → 只显示本楼层的 Ekahau AP
  - **列**（用户友好的中文表头）：AP Name / Vendor / Model / WiFi Standard / Frequency Bands / Mount Height (m) / Mount Type / Tx Power / Channels / MIMO Streams / Antenna / Tags
  - 按 Mark 升序排序
- **No** → 跳过

### 关键点

- **族类型选择被记忆**，多次运行不需要重选
- **暂存数据持久化**，可以随时再次运行 AP Place（但已放置的 AP 不会重复放）
- **WiFi 视图和明细表都可重用**：再次运行 AP Place 会复用已存在的同名视图/明细表，过滤器自动包含新 AP
- **批量事务**：20 个 AP 一个事务，失败的批次自动回滚但不影响其他批次

## 常见问题

### Q1：运行 Param Config 提示"未找到墙、门、窗"
**A**：当前活动视图不是楼层平面，或视图中确实没有这些元素。切换到正确的楼层平面视图后重试。

### Q2：ESX Export 时墙都用了 `Generic` 预设
**A**：说明这些墙类型没在 Param Config 中配置。先跑 Param Config 设置 `Ekahau_WallType` 参数，或在 ESX Export 的"映射检查"对话框里手动选择预设。

### Q3：ESX Read 后看不到临时标记
**A**：检查：
1. 楼层匹配对话框中是否选错了 Revit 视图
2. AP 确认对话框中是否取消了所有 AP 的勾选
3. 视图是否启用了详图项目（Detail Items）类别可见性

### Q4：AP Place 提示"未找到暂存数据"
**A**：必须先运行 ESX Read。如果运行过但提示项目不匹配，说明你打开的是另一个 Revit 项目（暂存数据按项目路径哈希隔离）。

### Q5：AP Place 后所有 AP 都堆叠在原点
**A**：可能是 ESX 文件的 Floor Plan 比例尺设置错误。检查 Ekahau 中楼层平面图的"Scale"设置，与 Revit 视图比例匹配后重新导出 .esx 再 ESX Read。

### Q6：WiFi 平面视图里看到一堆与 AP 无关的 Generic Model
**A**：这种情况下视图过滤器没生效。请确认：
1. 共享参数 `Ekahau_AP_Name` 已被正确创建（在 AP Place 步骤 4 自动完成）
2. 项目中的 Generic Models 类型本身没有显式设置 `Ekahau_AP_Name` 参数

### Q7：AP Schedule 中也出现了非 AP 的 Generic Models
**A**：同上，过滤器依赖 `Ekahau_AP_Name` 参数。在 AP Place 步骤 4 自动创建该参数；如果是手动创建的明细表，需要手工添加 `Ekahau_AP_Name has value` 过滤器。

### Q8：在 Revit 2023 上启动报错 "找不到 SpecTypeId"
**A**：v2.0.0 已通过反射兼容 Revit 2023 的 ParameterType。确认你安装的是 `net48` 版本的 DLL（位于 `Addins\2023\EkahauWiFiTools\`，而不是 net8 版本）。运行 `install.ps1` 会自动选对版本。

### Q9：Revit 版本对应的 DLL
| Revit 版本 | 运行时 | DLL 位置 |
|-----------|--------|----------|
| 2023, 2024 | .NET Framework 4.8 | `bin\Release\net48\` |
| 2025, 2026 | .NET 8 | `bin\Release\net8.0-windows\` |
| 2027 | .NET 10 | `bin\Release\net10.0-windows\` |

**重要**：Revit 2027 改用 .NET 10 运行时，net8 的 DLL **无法**在 Revit 2027 中加载。`install.ps1` 会自动选对版本。

### Q10：暂存文件位置
`%TEMP%\EkahauRevitPlugin\{项目名}_{12位哈希}\ap_staging.json`

不同项目自动隔离，可以放心同时打开多个项目。
